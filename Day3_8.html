Arguments in R

Arguments are always named when you define a function.

When you call a function, you do not have to specify the name of the argument.

Arguments are optional; you do not have to specify a value for them.

They can have a default value, which is used if you do not specify a value for that argument yourself.

You can use as many arguments as you like, there is no limit to the number of arguments.

An argument list comprises of comma-separated values that contain the various formal arguments

addPercent <- function(x, mult = 100, ...){ 
  percent <- round(x*mult, ...) 
  paste(percent, "%", sep = "") 
} 
percentages <- c(0.5823, 0.1214, 0.3349) 
addPercent(percentages)
## [1] "58%" "12%" "33%"
args(round) # show arguments of round
## function (x, digits = 0) 
## NULL
args(addPercent) # show arguments of addPercent
## function (x, mult = 100, ...) 
## NULL
addPercent(percentages, digits=1) 
## [1] "58.2%" "12.1%" "33.5%"

2 개의 요소로 구성된 함수를 만들어 보자.

2개의 요소로 구성된 함수의 경우 2개의 요소를 순서대로 입력하거나 혹은 요소의 이름과 함께 값을 입력하면 문제 없이 실행된다.

addPercent <- function(x, mult ){ 
  percent <- round(x*mult,  digits = 1)
  paste(percent, "%", sep = "")
} 

addPercent(percentages, mult=1)
## [1] "0.6%" "0.1%" "0.3%"
addPercent(percentages, 1)
## [1] "0.6%" "0.1%" "0.3%"
addPercent(mult=1, x=percentages)
## [1] "0.6%" "0.1%" "0.3%"

하지만 구성하는 요소의 값을 하나라도 생략시 오류가 발생한다.

addPercent(percentages) #error

그러면 mult 값을 입력하지 않더라도 오류가 발생하여 실행되도록 만들려면?

기본값을 설정해주면 된다.

기본값은 arguments의 요소를 입력하는 곳에 등호(=)와 함께 기본값(default value)를 설정해주자.

기본값이 설정된 경우에는 함수 실행시 생략되면 기본값을 실행시킨다.

addPercent <- function(x, mult=100){ 
  percent <- round(x*mult,  digits = 1)
  paste(percent, "%", sep = "")
} 

addPercent(percentages, mult=1)
## [1] "0.6%" "0.1%" "0.3%"
addPercent(percentages, 1)
## [1] "0.6%" "0.1%" "0.3%"
addPercent(mult=1, x=percentages)
## [1] "0.6%" "0.1%" "0.3%"
addPercent(percentages)
## [1] "58.2%" "12.1%" "33.5%"

Using the Dots arguments

나만의 함수를 만들어 실행하다보면 모든 arguments 요인을 명명하여 어려운 경우가 발생한다.

예를 들어 summary, lm, dplyr::filter 등의 함수의 arguments를 보면 …이 포함되어 있다.

`dplyr::filter` 에서 ::은 dplyr 패키지의 filter함수를 뜻한다.

여러 패키지를 이용하다보면 함수명이 동일한 경우가 있기 때문이 이를 구분하기 위한 방법이다.
args(summary) # 요약값 생성
## function (object, ...) 
## NULL
args(lm) # linear model 함수(회귀분석시 사용)
## function (formula, data, subset, weights, na.action, method = "qr", 
##     model = TRUE, x = FALSE, y = FALSE, qr = TRUE, singular.ok = TRUE, 
##     contrasts = NULL, offset, ...) 
## NULL
args(dplyr::filter)
## function (.data, ..., .preserve = FALSE) 
## NULL

더 상세한 정보가 필요한 경우에는 ?함수명 또는 help()함수를 이용하자.

Using functions as Arguments

R에서는 함수를 arguments로 사용할 수도 있다.

math.f <- function(x, FUN=mean,...){
  print(FUN(x,...))
}

x=c(1:5,NA,9:15)

math.f(x,sum)
## [1] NA
math.f(x,max, na.rm=T)
## [1] 15
math.f(x, na.rm=T)
## [1] 8.25

arguments는 함수를 구성하는 요소들을 의미하고 default, … 이 두가지만 잘 이해하시면 됩니다.
